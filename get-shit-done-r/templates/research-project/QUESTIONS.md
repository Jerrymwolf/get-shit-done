<guidelines>

**Central Questions:**
- Frame as genuine questions, not topics
- Include question type (descriptive, explanatory, evaluative, comparative)
- Note what would count as an answer — this drives source selection

**Sub-Questions:**
- Group by theme, not by source
- Assess tractability — some questions can't be answered with available evidence
- Note dependencies — some questions require others to be answered first

**Settled vs. Open:**
- Critical distinction — saves effort on settled questions
- "Settled" doesn't mean "true forever" — note the evidence basis
- "Open" means genuine expert disagreement, not "I don't know yet"

**Tractability:**
- Be honest about what can and can't be answered
- Source availability is a real constraint
- Low tractability doesn't mean unimportant — it means the project should acknowledge the gap

</guidelines>
