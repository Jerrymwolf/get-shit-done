   - Verify filenames follow generateSourceFilename convention

   - Log both successful acquisitions
   - Verify SOURCE-LOG.md has correct rows

   - Verify all referenced files exist
   - Verify no orphans
5. Run two-tier verification using verifyNote (verify-research.cjs)
   - Verify Tier 1 passes (note answers the research question)
   - Verify Tier 2 passes (all sources attached)

**Suite 2: Full pipeline — failure and fix task generation**
1. Create a note with missing source references
2. Run verifyNote — should fail at Tier 2
3. Call generateFixTasks — should produce acquisition fix tasks
4. Verify fix task descriptions are actionable

**Suite 3: State tracking across the pipeline**
1. Create a note and track in STATE.md using cmdStateAddNote (state.cjs)
2. Update note status through draft -> reviewed -> final
3. Add a source gap using cmdStateAddGap
4. Resolve the gap using cmdStateResolveGap
5. Verify STATE.md reflects all changes

All external calls (git, HTTP) mocked via dependency injection.
