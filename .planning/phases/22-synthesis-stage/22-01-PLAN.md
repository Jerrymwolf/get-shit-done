**grd-thematic-synthesizer.md:**
- Purpose: Identify patterns, themes, and relationships across all verified research notes. Construct themes through sustained engagement with evidence.
- Methodology: Braun & Clarke (2006) reflexive thematic analysis. Six phases: familiarization, generating initial codes, searching for themes, reviewing themes, defining and naming themes, producing the report. Themes are actively constructed, not passively discovered.
- Quality criteria: Every theme maps to specific notes via `[Note: {note-name}]` citations. No orphan notes (every note contributes to at least one theme). Contradictions between notes explicitly surfaced. Coverage: all research questions addressed by at least one theme.
- Output: Write to `{vault_path}/00-THEMES.md` using `grd/templates/themes.md` structure.

**grd-framework-integrator.md:**
- Purpose: Test accumulated evidence against the theoretical framework from Stage 1. Determine if the framework holds, needs modification, or should be replaced.
- Methodology: Carroll et al. (2013) best-fit framework synthesis. Use the a priori framework to organize evidence, then refine the framework based on findings. For interdisciplinary work, identify conflicts between disciplinary insights and produce integrated understanding.
- Inputs: THEMES.md + Theoretical Framework Survey note from Stage 1 + all verified notes.
- Quality criteria: Every framework element maps to supporting or contradicting evidence. Modifications to the original framework are justified with specific note citations. Framework is internally consistent.
- Output: Write to `{vault_path}/00-FRAMEWORK.md` using `grd/templates/framework.md` structure.

**grd-gap-analyzer.md:**
- Purpose: Systematically identify what is missing, contradicted, or assumed without justification in the research evidence.
- Methodology: Muller-Bloch & Kranz (2015) six gap types: contradictory evidence, knowledge voids, action-knowledge conflicts, methodological gaps, evaluation voids, theory application voids. PLUS Alvesson & Sandberg (2011) problematization: not "what hasn't been studied?" but "what has been assumed without justification?"
- Inputs: THEMES.md + FRAMEWORK.md + all verified notes.
- Quality criteria: Each gap typed using the Muller-Bloch & Kranz taxonomy. Each gap assessed for significance (high/medium/low). Problematization identifies at least one assumption challenge. Gaps link to specific notes and themes.
- Output: Write to `{vault_path}/00-GAPS.md` using `grd/templates/gaps.md` structure.

**grd-argument-constructor.md:**
- Purpose: Assemble thematic synthesis, theoretical integration, and gap analysis into a coherent scholarly argument. This is where the review becomes a contribution, not just a summary.
- Methodology: Read `deliverable_format` from PROJECT.md to select output structure. Baseline scholarly structure (D-10): Introduction (research questions + significance) -> Key Themes (from THEMES.md) -> Theoretical Implications (from FRAMEWORK.md) -> Gaps & Future Directions (from GAPS.md) -> Conclusion (the argument).
- deliverable_format options: `literature_review` (full academic structure), `research_brief` (condensed for practitioners), `build_spec` (technical specification for follow-on development), `custom` (researcher defines structure).
- Quality criteria: Inline note-level citations using `[Note: {note-name}]` format (D-11). Every claim links to supporting research note(s). All identified themes referenced. Argument advances beyond summary.
- Output: Write to `{vault_path}/00-Executive-Summary.md` using `grd/templates/executive-summary.md` structure. Markdown only (D-12).
