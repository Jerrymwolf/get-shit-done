- Top-level defaults: researcher_tier='standard', review_type='narrative', epistemological_stance='pragmatist'

- Deep-merge workflow: `{ ...smartDefaults, ...(rawConfig.workflow || {}) }` -- user's existing workflow keys win over smart defaults

- Preserve ALL other workflow keys (research, verifier, auto_advance, nyquist_validation, ui_phase, ui_safety_gate)
- Returns the updated config clone (does not mutate input)

**6. Add canDowngrade(currentType, requestedType) function:**
- Uses REVIEW_TYPE_ORDER.indexOf() for both
- Returns true only if requestedIdx > currentIdx (moving toward less rigor)
- Same type returns false (not a downgrade)
- Unknown types return false
