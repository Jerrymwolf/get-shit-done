# Plan 24-01: Wire Verification Pipeline and Fix Config Bug

## Goal

Make Tier 0 sufficiency checks reachable from the verification workflow via CLI, and fix the temporal_positioning config propagation bug.

## Context

### verify-sufficiency.cjs orphan
`verify-sufficiency.cjs` has full implementation (364 lines, 7 exported functions, 20 passing tests) but `grd-tools.cjs` never requires or routes to it. `verify-inquiry.md` instructs the agent to "Run structural sufficiency checks using verify-sufficiency.cjs module" but there is no CLI command to invoke it.

### temporal_positioning bug
`init.cjs:387` reads `config.workflow?.temporal_positioning` but `loadConfig()` returns `temporal_positioning` as a top-level property (not nested under `workflow`). The `?.` chain always evaluates to `undefined`, making `undefined !== false` always `true`. Era coverage checks never respect a disabled setting.

## Tasks

### Task 1: Add `verify sufficiency` subcommand to grd-tools.cjs

**File:** `grd/bin/grd-tools.cjs`

**Action:** In the `case 'verify':` block (line 393), add a new `else if` branch before the `else` fallback:

```javascript
} else if (subcommand === 'sufficiency') {
  const verifySufficiency = require('./lib/verify-sufficiency.cjs');
  const core = require('./lib/core.cjs');
  const config = core.loadConfig(cwd);
  const vaultPath = path.join(cwd, config.vault_path || '.planning/notes');
  const requirementsPath = path.join(cwd, '.planning', 'REQUIREMENTS.md');
  const result = verifySufficiency.verifySufficiency(vaultPath, requirementsPath, {
    reviewType: config.review_type || 'narrative',
    epistemologicalStance: config.epistemological_stance || 'pragmatist',
    temporalEnabled: config.temporal_positioning !== false && config.temporal_positioning !== 'optional',
  });
  const { output: outputFn } = require('./lib/core.cjs');
  outputFn(result, raw);
```

Also update the error message to include `sufficiency` in the available subcommands list.

**Commit:** `feat(phase-24): add verify sufficiency subcommand to grd-tools.cjs`

### Task 2: Fix temporal_positioning in cmdInitVerifyWork

**File:** `grd/bin/lib/init.cjs`, line 387

**Old:**
```javascript
temporal_positioning: config.workflow?.temporal_positioning !== false,
```

**New:**
```javascript
temporal_positioning: config.temporal_positioning !== false && config.temporal_positioning !== 'optional',
```

This reads the top-level property returned by `loadConfig()` and treats both `false` and `'optional'` as disabled (matching the SMART_DEFAULTS behavior where narrative/integrative/critical reviews set temporal_positioning to `'optional'`).

**Commit:** `fix(phase-24): read temporal_positioning from correct config path in init.cjs`
