<decisions>
## Implementation Decisions

### Two-Tier Verification (VERI-01, VERI-02, VERI-03)
- Tier 1 (goal-backward): "What must be TRUE for this research question to be answered?" — checks note quality
- Tier 2 (source audit): Confirms every reference has a file in -sources/, SOURCE-LOG accounts for all, no orphans
- Tiers run in order: goal-backward first, source audit second
- Implemented as a verification module (verify.cjs) that the grd-verifier agent loads
- Uses validateReferences() from acquire.cjs for Tier 2

### Verification Failure Handling (VERI-04)
- Failures automatically generate fix tasks via /grd:quick
- Fix tasks are specific: "acquire missing source X" or "strengthen analysis of Y"
- The grd-verifier agent creates fix task descriptions, /grd:quick executes them

### Orchestration Loop (ORCH-01, ORCH-02, ORCH-03)
- discuss/plan/execute/verify loop inherited from GSD — already functional via upstream commands
- Fresh 200K subagent contexts per task (inherited, unchanged)
- Wave parallelism for independent research tasks (inherited, unchanged)
- The /grd:* commands route to the same GSD workflows but with research-specific agents and templates

### Claude's Discretion
- How verify.cjs structures its checks internally
- Exact fix task generation format
- Whether to modify existing grd-verifier.md or create a new verification agent
- How /grd commands route to research-specific behavior
