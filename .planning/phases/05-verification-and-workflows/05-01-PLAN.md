<task type="auto">
  <name>Task 2: Implement verify-research.cjs</name>
  <files>grd/bin/lib/verify-research.cjs</files>
  <action>
Create grd/bin/lib/verify-research.cjs:

**verifyTier1(noteContent, researchQuestion)** — Goal-backward verification:
- Parse note content to find Key Findings, Analysis, Implications, Open Questions sections
- Derive conditions from research question (what must be TRUE to answer it)
- Check each condition against note content:
  - Key Findings section exists and is substantive (>50 chars, not placeholder)
  - Analysis section exists and is substantive (>100 chars, not placeholder)
  - Implications section exists
  - Content addresses the research question (keyword overlap check)
- Return { passed, conditions: [{condition, status, reason}], warnings: [] }

**verifyTier2(noteContent, sourcesDir, sourceLogPath)** — Source audit:
- Call validateReferences() from acquire.cjs for file/orphan checks
- Additional checks from research-verification.md:
  - Frontmatter `sources` count matches actual file count
  - SOURCE-LOG.md exists
  - SOURCE-LOG.md accounts for all referenced sources
  - No contradictions flagged without justification
- Return { passed, missing, orphans, issues: [{check, status, detail}] }

**verifyNote(noteContent, sourcesDir, sourceLogPath, researchQuestion)** — Combined:
- Run Tier 1 first
- If Tier 1 fails, return immediately with status 'draft' (don't run Tier 2)
- If Tier 1 passes, run Tier 2
- Determine status: final (both pass), reviewed (Tier 2 minor only), draft (any fail)
- Return { status, tier1Result, tier2Result }

**generateFixTasks(verificationResult, noteMetadata)** — Generate fix task descriptions:
- For Tier 1 failures: create research task description targeting the gap
- For Tier 2 missing sources: create acquisition task description
- For Tier 2 formatting issues: create cleanup task description
- Return array of { type, description, priority } objects

Import validateReferences and extractReferences from acquire.cjs.
Use only Node built-ins. No external dependencies.
Export all four functions from module.exports.
