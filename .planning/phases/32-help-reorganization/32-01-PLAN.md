**Section reorganization (HLP-01):**

### Research Workflow

Group commands by the natural research lifecycle:

**Scoping and Planning:**
- `/grd:new-research` — Define research questions, explore the landscape (spawns 4 parallel researcher agents), scope requirements, create a roadmap of inquiry phases
- `/grd:scope-inquiry <N>` — Articulate your vision for a line of inquiry before planning (captures methodology choices, scope boundaries, theoretical emphasis)
- `/grd:research-phase <N>` — Deep ecosystem research for specialized domains (discovers standard approaches, frameworks, methodological pitfalls)
- `/grd:list-phase-assumptions <N>` — Preview Claude's intended approach before execution
- `/grd:plan-inquiry <N>` — Create detailed execution plan for a line of inquiry

**Conducting Research:**
- `/grd:conduct-inquiry <N>` — Execute all plans in an inquiry phase (parallel subagents, verification on completion)
- `/grd:map-corpus` — Survey existing sources and knowledge landscape (produces 7 structured corpus documents: SOURCES, DOMAINS, METHODOLOGY, COVERAGE, GAPS, CONNECTIONS, QUALITY)
- `/grd:add-verification <N>` — Add evidence checks, source coverage assertions, and methodology validation for a completed inquiry phase
- `/grd:diagnose [issue]` — Investigate methodology gaps, source conflicts, and analytical dead ends with persistent diagnosis sessions
- `/grd:verify-inquiry <N>` — Validate research findings through structured acceptance testing

**Synthesis and Delivery:**
- `/grd:synthesize` — Transform verified notes into structured scholarship (themes, frameworks, gaps, argument construction)
- `/grd:presentation-design [N]` — Plan how findings will be presented (paper, poster, slide deck, report) — creates PRESENTATION-SPEC.md
- `/grd:output-review [N]` — Audit quality of research deliverables across six scholarly dimensions: argument coherence, evidence quality, citation coverage, methodology transparency, presentation clarity, completeness
- `/grd:export-research [N]` — Package completed research for delivery (Obsidian vault, manuscript assembly, shareable archive)
- `/grd:export-clean [dir]` — Create clean research package without .planning/ artifacts for sharing or submission

**Inquiry Management:**
- `/grd:add-phase <desc>` — Add a new line of inquiry to the current milestone
- `/grd:insert-phase <after> <desc>` — Insert urgent inquiry between existing phases
- `/grd:remove-phase <N>` — Remove a future inquiry phase and renumber
- `/grd:new-milestone <name>` — Start a new research milestone (questioning, research, requirements, roadmap)
- `/grd:complete-study <version>` — Archive completed milestone, create git tag, prepare for next

### Smart Router

- `/grd:do <description>` — Route freeform text to the right GRD command automatically

### Quick Mode

- `/grd:quick [--full] [--discuss] [--research]` — Execute small ad-hoc research tasks with abbreviated pipeline
- `/grd:fast [description]` — Execute trivial tasks inline (no subagents, no planning files)

### Utility

- `/grd:progress` — Check research progress and route to next action
- `/grd:stats` — Display project statistics (phases, plans, requirements, git metrics, timeline)
- `/grd:health [--repair]` — Diagnose .planning/ directory integrity and optionally repair issues
- `/grd:resume-work` — Resume research from previous session with full context restoration
- `/grd:pause-work` — Create context handoff when pausing mid-inquiry
- `/grd:note <text>` — Zero-friction idea capture (append, list, promote)
- `/grd:add-todo [desc]` — Capture research task or idea as structured todo
- `/grd:check-todos [area]` — Review pending todos and select one to work on
- `/grd:plant-seed [idea]` — Capture forward-looking idea with trigger conditions for automatic surfacing
- `/grd:review --phase N [--all]` — Cross-AI peer review of inquiry plans
- `/grd:audit-milestone [version]` — Audit milestone completion against original intent
- `/grd:plan-milestone-gaps` — Create phases to close gaps from milestone audit
- `/grd:audit-uat` — Cross-phase audit of outstanding verification items
- `/grd:cleanup` — Archive phase directories from completed milestones
- `/grd:join-discord` — Join the GRD community

### Configuration

- `/grd:settings` — Configure workflow toggles and model profile interactively
- `/grd:set-profile <profile>` — Quick switch model profile (quality/balanced/budget/inherit)
- `/grd:update` — Update GRD to latest version with changelog preview
- `/grd:help` — Show this reference
