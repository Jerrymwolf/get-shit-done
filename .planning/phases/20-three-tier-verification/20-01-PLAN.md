    Create `grd/bin/lib/verify-sufficiency.cjs` following the CJS module pattern from plan-checker-rules.cjs:

    1. **SUFFICIENCY_CRITERIA lookup table** (mirrors RIGOR_LEVELS pattern):
       ```
       systematic: { min_notes_per_objective: 3, require_primary_sources: true, require_methodological_diversity: true, min_eras: 3, eras_required: true }
       scoping: { min_notes_per_objective: 1, require_primary_sources: false, require_methodological_diversity: false, min_eras: 2, eras_required: true }
       integrative: { min_notes_per_objective: 1, require_primary_sources: false, require_methodological_diversity: false, min_eras: 0, eras_required: false }

       ```

    2. **discoverNotes(vaultDir)**: Use `fs.readdirSync` with `withFileTypes` to walk vault subdirectories. For each `.md` file (excluding SOURCE-LOG*), call `parseFrontmatter()` from verify-research.cjs. Include notes that have `inquiry` or `domain` frontmatter fields. Return `[{ path, frontmatter, keyFindings }]` where keyFindings is extracted via `extractSection(content, 'Key Findings')`.

    3. **parseObjectives(requirementsContent)**: Regex match `- \[[ x]\] \*\*([^*]+)\*\*:\s*(.+)` to extract objective IDs and descriptions. Return `[{id, description}]`.

    4. **checkObjectiveCoverage(notes, objectives, reviewType)**: For each objective, find matching notes. Primary match: note's `inquiry` frontmatter field matches objective's phase context. Fallback: keyword overlap between note's keyFindings and objective description using `extractKeywords()` from verify-research.cjs. Compare count against `SUFFICIENCY_CRITERIA[reviewType].min_notes_per_objective`. Return `{ sufficient: boolean, gaps: [{type:'objective_coverage', objective, description, found, required}] }`.

    5. **checkEraCoverage(notes, reviewType, temporalEnabled)**: If !temporalEnabled, return `{sufficient:true, gaps:[], eraDistribution:{}, skipped:true}`. Count notes per era from frontmatter `era` field. Valid eras: `['foundational', 'developmental', 'contemporary', 'emerging']`. Compare distinct eras present against `SUFFICIENCY_CRITERIA[reviewType].min_eras`. Return `{ sufficient, gaps, eraDistribution }`.

    6. **checkMethodologicalDiversity(notes, objectives, reviewType)**: Only enforced when `SUFFICIENCY_CRITERIA[reviewType].require_methodological_diversity` is true (systematic only). For each objective, check that its matching notes reference more than one methodological approach (check note frontmatter for `methodology` field or keywords in Key Findings like "qualitative", "quantitative", "mixed-methods", "case study", "survey", "experiment", "meta-analysis", "interview", "ethnography"). Return `{ sufficient, gaps }`.

    7. **checkEpistemologicalConsistency(notes, stance)**: If stance is 'pragmatist', return `{consistent:true, warnings:[], reason:'Pragmatist stance: methodological flexibility expected'}`. For other stances, return `{consistent:true, warnings:[]}` as a data-only stub -- the actual qualitative assessment is done by the agent. The CJS module only provides the auto-pass logic for pragmatist.

    8. **verifySufficiency(notes, objectives, config)**: Orchestrate all checks. Extract `reviewType` from config.review_type (default 'narrative'), `temporalEnabled` from config.workflow?.temporal_positioning !== false. Combine all gaps. Return `{ sufficient: allGaps.length === 0, gaps, summary: { total_notes, objectives_checked, review_type, era_distribution, methodological_diversity, epistemological_consistency } }`.
