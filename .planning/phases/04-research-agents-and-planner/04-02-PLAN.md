**grd/bin/lib/bootstrap.cjs:**

parseBootstrap(content) — Parse BOOTSTRAP.md into structured data:
- Extract rows from each tier's table (Established, Partially Established, Not Yet Researched)
- Return { established: [...], partial: [...], notResearched: [...] }

generateBootstrap(findings) — Generate BOOTSTRAP.md content from findings:
- Read template, fill tables with provided findings
- Return markdown string

queryBootstrap(content, finding) — Check if a finding is already established:
- Parse bootstrap, search across tiers
- Return { found: boolean, tier: string, details: object }

**grd/bin/lib/plan-checker-rules.cjs:**

checkSourceDuplication(planContent, bootstrapContent) — Check for re-researching known findings:
- Parse bootstrap, extract established findings
- Check if plan tasks overlap with established findings
- Return { valid, issues }

checkSourceLimits(planContent) — Enforce max 3 sources per task:
- Parse research task XML, count <src> blocks per task
- Return { valid, issues }

checkAcquisitionMethods(planContent) — Verify all sources have methods:
- Parse <src> blocks, check method attribute exists and is valid
- Valid methods: firecrawl, web_fetch, wget, gh-cli
- Return { valid, issues }

checkContextBudget(planContent) — Verify >30 page papers get dedicated tasks:
- Parse tasks, check if any multi-source task references papers noted as >30 pages
- This relies on source notes in the plan (e.g., "28 pages" in notes column)
- Return { valid, issues }

validateResearchPlan(planContent, bootstrapContent) — Run all checks:
- Call all four checkers, aggregate results
- Return { valid, issues: [...all issues] }

Both modules use zero npm dependencies. Export all functions.
