Add the following functions to state.cjs:

**Note Status Functions:**
- `cmdStateAddNote(cwd, options, raw)` — Add or update a note in the Note Status table
  - options: { note, status, sources, date }
  - status must be one of: draft, reviewed, final, source-incomplete
  - If note already exists in table, update its row instead of adding duplicate
- `cmdStateUpdateNoteStatus(cwd, note, status, raw)` — Update just the status of an existing note
- `cmdStateGetNotes(cwd, raw)` — Parse and return all notes from the Note Status table

**Source Gap Functions:**
- `cmdStateAddGap(cwd, options, raw)` — Add a gap to the Source Gaps table
  - options: { note, source, reason, impact }
- `cmdStateResolveGap(cwd, note, source, raw)` — Remove a specific gap (by note + source match)
- `cmdStateGetGaps(cwd, raw)` — Parse and return all gaps from the Source Gaps table

Implementation patterns (follow existing state.cjs conventions):
- Use regex to find the section heading and table
- Parse table rows into structured data
- Modify table rows and write back
- Use writeStateMd() for all writes (keeps frontmatter in sync)
- Use output() for JSON results, error() for failures
- Follow the cmd* naming convention

Table parsing approach:
- Find `## Note Status` or `## Source Gaps` heading
- Skip header row and separator row
- Parse data rows by splitting on `|`
- When adding: append new row
- When updating: find matching row by note name, replace
- When resolving: find matching row, remove

Export all new functions from module.exports.
