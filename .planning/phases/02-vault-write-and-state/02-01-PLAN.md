Add atomicWrite() function to vault.cjs. Implementation approach:

1. Track created artifacts in an array for rollback
2. Use existing writeNote() for note file + sources dir
3. Read SOURCE-LOG.md template from templates/ directory (resolve path relative to __dirname)
4. Write SOURCE-LOG.md into the sources directory
5. Run git add and git commit using execFileSync (NOT exec — no shell injection risk)
   - Use child_process.execFileSync('git', ['add', ...files], { cwd, stdio: 'pipe' })
   - Use child_process.execFileSync('git', ['commit', '-m', message], { cwd, stdio: 'pipe' })
   - Accept an optional `gitRunner` function for dependency injection in tests
6. Extract commit hash from git output
7. On any failure, delete all created artifacts in reverse order

Key design decisions:
- Uses try/catch with artifact tracking for rollback
- Reuses existing writeNote() for note + sources dir creation
- Git operations use execFileSync for safety (no shell expansion)
- Accepts options.gitRunner for test mockability
- Accepts options.cwd for git working directory and options.topic for commit message
- Returns { notePath, sourcesDir, sourceLogPath, commitHash }

Export atomicWrite from module.exports.
