## Cross-Phase Integration Report

### CRITICAL: Init Subcommand Name Mismatch

Phase 17 renamed init subcommands in `grd-tools.cjs` to research-native vocabulary, but 8 workflow files still call the old names. Every call hits the `default` error case.

| Workflow | Calls | CLI Expects |
|----------|-------|-------------|
| `plan-inquiry.md:18` | `init plan-phase` | `init plan-inquiry` |
| `conduct-inquiry.md:19` | `init execute-phase` | `init conduct-inquiry` |
| `verify-inquiry.md:30` | `init verify-work` | `init verify-inquiry` |
| `new-research.md:49` | `init new-project` | `init new-research` |
| `execute-plan.md:18` | `init execute-phase` | `init conduct-inquiry` |
| `complete-study.md:597` | `init execute-phase` | `init conduct-inquiry` |
| `synthesize.md:18` | `init execute-phase` | `init conduct-inquiry` |
| `ui-phase.md:16` | `init plan-phase` | `init plan-inquiry` |

**Root cause:** Phase 17 renamed CLI cases but did not update workflow callers in the same atomic change.

### HIGH: verify-sufficiency.cjs Not Exposed via CLI

`verify-sufficiency.cjs` has full implementation (364 lines, 20 tests) but `grd-tools.cjs` never requires or routes to it. `verify-inquiry.md` instructs the agent to "Run structural sufficiency checks using verify-sufficiency.cjs module" but provides no CLI mechanism.

### MEDIUM: temporal_positioning Bug

`init.cjs:387` reads `config.workflow?.temporal_positioning` but `loadConfig()` returns `temporal_positioning` as a top-level property. The `?.` chain evaluates to `undefined`, making `undefined !== false` always `true`. Era coverage checks never respect a disabled temporal_positioning setting.

### LOW: tier-strip.cjs Zero Runtime Consumers

`tier-strip.cjs` is only consumed by tests. Tier-conditional content in workflows and templates is processed by the LLM reading XML/comment blocks directly, not by CJS code at runtime. May be intentional — module exists for potential future programmatic use.
