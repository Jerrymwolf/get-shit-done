# Requirements: GSD-R

**Defined:** 2026-03-11
**Core Value:** Every research finding is self-contained and auditable — the note plus its actual source files, never just links

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Fork Setup

- [x] **FORK-01**: GSD repo cloned from https://github.com/gsd-build/get-shit-done and restructured as GSD-R
- [x] **FORK-02**: All `/gsd:*` commands renamed to `/gsd-r:*` with clean namespace separation
- [x] **FORK-03**: Package renamed and scoped for eventual npm publication (`@gsd-r/get-shit-done-r` or similar)
- [x] **FORK-04**: Git LFS configured for binary files (PDFs, images) before first binary commit

### Foundation

- [x] **FOUN-01**: Filesystem vault write — notes and sources written to disk at configurable `vault_path`
- [x] **FOUN-02**: File naming convention enforced: `{descriptive-slug}_{date-acquired}.{ext}` for all source files
- [x] **FOUN-03**: Research note template with frontmatter (project, domain, status, date, sources count) and fixed sections (Key Findings, Analysis, Implications, Open Questions, References)
- [x] **FOUN-04**: `vault_path` configurable in config.json
- [x] **FOUN-05**: Auto-commit research notes to git — vault write + git commit as atomic pair, versioning the note + sources folder + SOURCE-LOG.md together

### Source Management

- [x] **SRC-01**: Source Attachment Protocol — every cited source has a corresponding file in the note's `-sources/` folder
- [x] **SRC-02**: Source acquisition fallback chain: firecrawl → web_fetch → wget/curl → mark unavailable in SOURCE-LOG.md
- [x] **SRC-03**: SOURCE-LOG.md generated per note documenting every acquisition attempt (source, URL, method, file, status, notes)
- [x] **SRC-04**: Unavailable sources documented but do not block task completion — gap flagged in Open Questions
- [x] **SRC-05**: Sources acquired in original form — PDFs as PDFs, not converted; HTML scraped to markdown preserving structure; screenshots as PNG
- [x] **SRC-06**: Finding-to-source traceability — every claim in a note's Analysis section links to a specific local source file in `-sources/` (not a URL), so the reader can open the original and verify the claim themselves
- [x] **SRC-07**: Executive Summary and synthesis notes trace findings back to individual research notes, which in turn trace to source files — two-hop verifiability from any claim to the original document

### Orchestration

- [x] **ORCH-01**: Discuss/plan/execute/verify loop inherited from GSD and functional for research workflows
- [x] **ORCH-02**: Fresh 200K subagent contexts per task (inherited, unchanged)
- [x] **ORCH-03**: Wave parallelism for independent research tasks (inherited, unchanged)
- [x] **ORCH-04**: Research-specific subagents: source researcher, methods researcher, architecture researcher, limitations researcher
- [x] **ORCH-05**: Research task XML template with `<src method="" format="">` blocks and research-oriented `<verify>` criteria
- [x] **ORCH-06**: Modified plan-checker validates: no duplication of BOOTSTRAP.md findings, primary sources preferred, ≤3 sources per task, acquisition method specified
- [x] **ORCH-07**: Context-budget-aware task splitting enforced — papers >30 pages get dedicated task, each task fits ~50% subagent context

### Knowledge Management

- [x] **KNOW-01**: BOOTSTRAP.md generated during new-project with three tiers: Established / Partially Established / Not Yet Researched
- [x] **KNOW-02**: STATE.md extended with note-status tracker (draft / reviewed / final / source-incomplete)
- [x] **KNOW-03**: STATE.md extended with source-gap reporting table (note, missing source, reason, impact)

### Verification

- [x] **VERI-01**: Goal-backward verification (Tier 1) — "What must be TRUE for this research question to be answered?"
- [x] **VERI-02**: Source audit verification (Tier 2) — every reference has a file in `-sources/`, SOURCE-LOG.md accounts for all sources, no orphan files
- [x] **VERI-03**: Two tiers run in order: goal-backward first, source audit second
- [x] **VERI-04**: Verification failures generate fix tasks via `/gsd-r:quick`

### Output

- [x] **OUTP-01**: Decision Log format — Decision, Options Considered, Evidence (note reference), Chosen, Rationale, Reversible?, Date
- [x] **OUTP-02**: Terminal deliverable option — Executive Summary in GSD `--auto` input format (What I Want Built / Project Context / Milestones / Tool Usage / Verification Criteria)

### End-to-End Validation

- [x] **E2E-01**: One full research project completed end-to-end using GSD-R: new-project → discuss → plan → execute → verify across at least 2 phases

## Out of Scope

| Feature | Reason |
|---------|--------|
| Automatic paper summarization | Value is in directed synthesis against research questions, not bulk summarization |
| Obsidian MCP as required dependency | Filesystem default ensures CLI works headless without external apps |
| Real-time collaboration / multi-user | Massively complex for v1; git branches suffice for team workflows |
| Web UI or TUI dashboard | CLI-native; STATE.md is the dashboard |
| Full-text search across vault | grep, VS Code, and Obsidian already do this; not GSD-R's job |
| Citation formatting (APA, MLA) | Not an academic writing tool; simple reference format suffices |
| RAG / search engine over vault | Separate project; massive scope creep |
| Browser extension for source capture | Separate product with separate maintenance cycle |
| Scheduled source re-checking | Sources are physically attached; URL death is irrelevant once acquired |
| Auto-commit to git | Moved back to v1 (FOUN-05) — essential for source file versioning and auditability |
