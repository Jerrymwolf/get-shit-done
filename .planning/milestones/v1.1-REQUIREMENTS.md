# Requirements: GSD-R v1.1 Upstream Sync

**Defined:** 2026-03-15
**Core Value:** Every research finding is self-contained and auditable -- the note plus its actual source files, never just links

## v1.1 Requirements

Requirements for upstream sync (GSD v1.22.4 -> v1.24.0). Each maps to roadmap phases.

- [x] **FOUN-01**: Create model-profiles.cjs with GSD-R agent entries (gsd-r-source-researcher, gsd-r-verifier, etc.)

- [x] **FOUN-03**: Add milestone scoping utilities to core.cjs (getMilestonePhaseFilter, stripShippedMilestones, replaceInCurrentMilestone)
- [x] **FOUN-04**: Add VERSION file tracking upstream base version (1.24.0)
- [x] **FOUN-05**: Tests verify model profile resolution, inheritance, and milestone filtering

### Core Library Merges

- [x] **CORE-01**: Sync core.cjs with upstream v1.24.0 while preserving GSD-R-specific logic
- [x] **CORE-02**: Sync state.cjs with upstream v1.24.0 while preserving Note Status and Source Gaps functions
- [x] **CORE-03**: Sync commands/routing with upstream (add new CLI subcommands for stats, autonomous, etc.)
- [x] **CORE-04**: Sync install.js with upstream changes
- [x] **CORE-05**: All existing tests pass against merged code (vault, acquire, state, bootstrap, plan-checker-rules, verify-research, e2e)

### Execution Rigor

- [x] **EXEC-02**: Update execute-plan.md with mandatory read_first gate and acceptance_criteria validation
- [x] **EXEC-03**: Add node-repair.md workflow with source-acquisition-aware failure diagnosis
- [x] **EXEC-04**: Add autonomous.md workflow with research-domain detection in smart discuss
- [x] **EXEC-05**: Add stats.md workflow with research-specific metrics (note count, source coverage, source gaps)

### Workflow Sync

- [x] **WKFL-02**: Add UI-SPEC.md template and copilot-instructions.md template
- [x] **WKFL-03**: Sync existing workflow files with upstream refinements (~25 overwrite files)
- [x] **WKFL-04**: Sync existing command files with upstream refinements

### Templates and Agents

- [x] **TMPL-01**: Sync shared agent prompts with upstream improvements (executor, verifier, codebase-mapper, debugger)
- [x] **TMPL-02**: Sync template files with upstream refinements (VALIDATION.md, debug templates, etc.)

## Future Requirements

### Performance Optimization

- **PERF-01**: Research vault indexing for faster note lookup in large vaults

## Out of Scope

| Feature | Reason |
|---------|--------|
| UI-SPEC for research notes | Research notes are not UI -- category confusion |
| Smart discuss vault scanning | BOOTSTRAP.md is the prior-knowledge gate; vault grep risks context bloat |
| Upstream features beyond v1.24.0 | Sync target is v1.24.0 specifically |
| New GSD-R research features | v1.1 is sync-only; new research features go in v1.2+ |
