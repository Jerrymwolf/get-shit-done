2. **config-set-model-profile stub** (INFO): gsd-r-tools.cjs lines 486-489 errors instead of routing. Documented intentional stub — upstream config.cjs sync pending. Affects CORE-03 peripherally.

3. **2 stale Skill() namespace calls** (MEDIUM): `plan-phase.md:529` uses `Skill(skill="gsd:execute-phase")` and `discuss-phase.md:682` uses `Skill(skill="gsd:plan-phase")`. Both should use `gsd-r:` prefix. `autonomous.md` correctly uses `gsd-r:` for all its Skill() calls. These will fail to resolve at runtime. Affects WKFL-03.

4. **Command frontmatter `name: gsd:`** (INFO): All 35 command files use `name: gsd:<command>` in frontmatter, which is the established GSD-R convention. Claude Code derives slash command namespace from directory structure (`commands/gsd-r/`), so runtime behavior is correct. Documented as intentional in Phase 13 SUMMARYs.

5. **Orphaned utility exports** (INFO): `VALID_PROFILES`, `formatAgentToModelMapAsTable`, `getAgentToModelMapForProfile` exported from model-profiles.cjs but not imported by core.cjs or commands.cjs. Benign — available for external consumers.

6. **replaceInCurrentMilestone zero call sites** (INFO): Exported from core.cjs but has no consumers anywhere in the codebase. Forward-looking API from upstream. Affects FOUN-03 peripherally.
